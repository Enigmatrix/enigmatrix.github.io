Dark Ghidra Setup

How to setup Ghidra to have a eye-pleasing dark mode
ghidraconfig

I usually do most of my work during (and throughout) the night, which includes using Ghidra (opens new window) for exploiting pwnable.xyz (opens new window) binaries.

Unfortunately, the default scheme of Ghidra is a light scheme which is quite glaring in the darkness of my room.

Ghidra does have its own 'dark mode', but it doesn't align with the colorscheme with the rest of my toolset, which are usually variants of Solarized (opens new window) or the Challenger Deep (opens new window) theme. Furthermore, it is not the most compatible or eye-pleasing.

# Solution

So I began to look for a way to use a better theme and ended up with this Github issue (opens new window), which shows how to use the Jtattoo (opens new window) Look and Feel (LaF). However, it's kinda old and unrefined, so I ended searching for other LaFs, and ended using FlatLaf (opens new window), which hit the mark for me.

TIP

If you are using Windows instead of Mac/Linux, then look into Ghidracula (opens new window). It doesn't work on Linux, which is why I ended using alternative methods such as these. There are also LaFs that might work for you, such as Quaqua (opens new window) if you want that Mac-ish look or WebLaf (opens new window).

# Installing LaFs

The instructions for using a certain LaF for Ghidra are in the Github issue above, but I will give a rehashing here for FlatLaf.

  1. Download the jar file for flatlaf-0.18.jar from here (opens new window) and place it somewhere.
  2. Place the path to the jar in the highlighted lines below in <GHIDRA_PATH>/support/launch.sh





 






 




SUPPORT_DIR="${0%/*}"
if [ -f "${SUPPORT_DIR}/launch.properties" ]; then

	# Production Environment
	INSTALL_DIR="${SUPPORT_DIR}/.."
	CPATH="${INSTALL_DIR}/Ghidra/Framework/Utility/lib/Utility.jar:<PATH_TO_JAR>"
	LS_CPATH="${SUPPORT_DIR}/LaunchSupport.jar"
	DEBUG_LOG4J="${SUPPORT_DIR}/debug.log4j.xml"
else

	# Development Environment
	INSTALL_DIR="${SUPPORT_DIR}/../../../.."
	CPATH="${INSTALL_DIR}/Ghidra/Framework/Utility/bin/main:<PATH_TO_JAR>"
	LS_CPATH="${INSTALL_DIR}/GhidraBuild/LaunchSupport/bin/main"
	DEBUG_LOG4J="${INSTALL_DIR}/Ghidra/RuntimeScripts/Common/support/debug.log4j.xml"
fi
  1. Set FlatLaf as the systemlaf. Add this line to <GHIDRA_PATH>/support/launch.properties.
VMARGS=-Dswing.systemlaf=com.formdev.flatlaf.FlatDarkLaf
  1. Use the System LaF in Ghidra, then restart Ghidra and enjoy! 🎉
